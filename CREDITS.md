## What "Inspired By" Means

We read these skills, learned from their approaches, and wrote our own versions from scratch. Specifically:

- **From debugging-strategies:** The 4-phase cycle structure and scientific method emphasis. We added the ACH methodology, delegation patterns, and Claude Code-specific workflow.
- **From maestro-delegation-advisor:** The concept of score-based delegation decisions. We replaced the 3-agent focus with a universal framework, added the 6-section prompt template, and model selection matrix.
- **From writing-plans + executing-plans:** The TDD-first plan structure and batch execution with checkpoints. We merged two separate skills into one coherent flow and added Quick DSV, deviation handling, and blocker protocols.
- **From reviewing-pull-requests:** The severity categorization framework and worktree isolation pattern. We added self-review quick start, delegation pattern for large PRs, and the security-first approach.
- **From ClaudeForge:** The tiered loading concept and configuration scoring patterns. We dropped all Python tooling in favor of pure markdown guidance and added the starter template and token measurement guide.
